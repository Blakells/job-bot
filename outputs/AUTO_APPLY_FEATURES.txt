COMPLETE FEATURE LIST:
----------------------

1. FILE UPLOAD SUPPORT
   - Automatically finds resume in uploads/ folder
   - PDF prioritization (falls back to .docx)
   - Handles file input fields with is_file_upload flag
   - Uses set_input_files() for reliable upload

2. MULTI-PAGE NAVIGATION
   - Detects "Continue" buttons vs "Submit" buttons
   - Tracks page numbers for screenshots
   - Loops through form pages until final submission
   - Claude detects has_continue_button with text

3. REACT-SELECT DROPDOWN HANDLING
   - EXACT working code from test_dropdown_direct.py
   - Finds label → gets "for" attribute → finds combobox by ID
   - Clicks combobox to open
   - Finds option by text (div[class*='option'] or role='option')
   - Clicks option to select
   - Handles is_react_select flag

4. LOCATION AUTOCOMPLETE
   - Uses .type() instead of .fill() for React autocomplete
   - Types city name with delay (e.g. "Florence")
   - Waits 1.5s for suggestions to load
   - Clicks first visible option
   - Special handling for #candidate-location field

5. ALL MISSING TEXT FIELDS
   - "How did you hear about this job?" → LinkedIn (from extra_answers)
   - "LinkedIn URL/Profile" → uses personal.linkedin_url
   - "Do you know anyone who works at..." → No (from extra_answers)
   - "Full legal name" → uses personal.name
   - All fields pulled from profile/job_profile.json

WORKING PATTERN FROM TEST:
--------------------------
1. Find label element by text
2. Get label's "for" attribute
3. Find input/combobox by ID
4. Click to open dropdown
5. Wait for options to appear
6. Find option by text
7. Click option

This exact pattern is used in fill_react_select_dropdown()
